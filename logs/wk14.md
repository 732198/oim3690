## Projects I checked out

### My Project — Pest Prediction Risk Analyzer
A tool that takes a user's location, fetches real-time weather data (temperature, humidity, precipitation) from an open-source weather API, and passes that environmental context to the Claude API to generate a pest risk assessment. The goal was to turn raw weather data into an actionable, human-readable risk analysis — answering not just "what's the weather" but "what does this weather mean for pest activity." The API chaining pattern (weather data → Claude prompt → risk output) was the core technical challenge of the project.

### CalorieCraft — AI-Powered Weekly Meal Planner
One of my favorites from Demo Day. You enter a daily calorie target and a dietary preference (anything, balanced, high protein, low carb, or vegetarian), and the app generates a full 7-day meal plan with breakfast, lunch, and dinner. It also produces a grocery list automatically and lets you save favorite plans locally. The UI was polished and the three-step flow (set goal → generate → save & shop) made it feel like a real product. No sign-up required, which lowered the barrier to try it immediately.

### Financial Freedom — Personal Finance Beginner Guidebook
A clean, jargon-free multi-page guide covering budgeting, debt payoff strategies, saving, investing, spending, and retirement. What stood out was how well-scoped it was — instead of trying to be a full financial app, it focused on clarity and confidence for beginners. The "Your First 3 Steps" onboarding section (review last month, set one goal, check in weekly) was a nice touch that made it immediately actionable rather than just informational.


## Key takeaways

- Seeing everyone's projects in one place made it clear how much range is possible with the same core tools — HTML, CSS, JavaScript, and an API; the difference between projects was mostly about idea clarity and scope, not technical complexity
- The best projects solved a specific, real problem for a specific person — CalorieCraft for someone overwhelmed by meal planning, Financial Freedom for someone intimidated by personal finance, pest prediction for someone making decisions based on environmental risk
- Chaining two APIs (weather + Claude) added real value that neither could provide alone; the weather API gave factual data and Claude gave interpretation — that division of labor is a useful pattern going forward
- A gallery walk is a much better format for evaluating web apps than a traditional presentation — you actually use the thing instead of just watching someone talk about it
- Deployment being a hard requirement meant every project was genuinely live and testable, not just screenshots

## Reflection

Demo Day was a good way to end the semester. Trying other people's projects as actual users — typing in a calorie goal, clicking through a finance guide — made the work feel real in a way that a slideshow presentation wouldn't have. CalorieCraft and Financial Freedom both stuck with me because they were immediately useful and clearly built around someone's genuine frustration with an existing problem, which is exactly what the course kept pushing us toward when brainstorming project ideas.

For my own project, Demo Day was a useful stress test. Having strangers try your app surfaces edge cases and UX issues that you stop noticing after building it for weeks. The pest prediction concept got good reactions — people understood the value quickly, which meant the framing worked even if there were rough edges in the implementation.

Looking back at the full semester, the progression made sense: HTML and CSS gave you structure and style, JavaScript gave you interactivity, APIs gave you real data, and the final project was the place where all of it had to work together at once. The hardest skill wasn't any single concept — it was learning to think about what a user actually needs and building toward that instead of just building what was technically interesting.
