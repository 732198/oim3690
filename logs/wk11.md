### Session 21 — Intro to APIs
- **What is an API** — a server that responds to HTTP requests with data, just like a browser does
- **JSON** — the data format APIs return; looks like a JS object but is actually just text that gets parsed
- **`fetch()`** — sends an HTTP request from JavaScript and returns a Promise
- **`async/await`** — syntax that lets you write asynchronous code in a readable, top-to-bottom style; `await` pauses execution until the Promise resolves
- **`try/catch`** — wraps fetch calls to handle errors gracefully instead of crashing
- Live demo using the OIM Teaching API (`oim.108122.xyz`)


## Key takeaways

- A `for` loop running `i < 10` executes **10 times** (i = 0 through 9); the last value of `i` inside the loop is 9
- If a sum loop always returns 0, the variable was probably declared inside the loop instead of outside — it resets every iteration
- `for...of` gives you the value directly; classic `for` gives you the index — use `for...of` when you don't need `i`
- `fetch()` is asynchronous — it doesn't return data immediately, it returns a Promise
- `await` only works inside an `async` function; forgetting `async` is the most common fetch bug
- `.json()` is also async and must be awaited — it converts the raw response text into a usable JS object
- `try/catch` around fetch handles network errors and bad responses so the page doesn't break
