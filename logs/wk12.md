## Key takeaways

- Never hardcode an API key directly in your HTML or JS files — if it gets pushed to GitHub, it's exposed publicly and can be abused
- `.gitignore` tells Git to completely ignore a file; adding `config.js` there means the key stays local while everything else gets committed normally
- The OpenAI API uses a POST request, not GET — you send a JSON body with the model name, a messages array, and your prompt
- A loading message ("Thinking...") is essential UX for any API call that might take a few seconds to respond
- The Final Project needs at least 3 sections/views, responsive design, at least 3 interactive features, and a working deployed URL
- The best project ideas come from a real frustration, something you'd actually use, or a problem someone around you has
